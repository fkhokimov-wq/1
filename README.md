# EIP Grant Management System

**Система управления грантовыми заявками** — веб-приложение для полного цикла обработки грантовых заявок: от подачи заявки до мониторинга выданного гранта.

## Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Frontend | Vanilla JavaScript (ES5+) |
| Стилизация | Tailwind CSS (CDN) |
| Иконки | Lucide Icons (CDN) |
| Шрифты | Google Fonts — Inter |
| Персистентность | In-memory (состояние в `window.state`) |
| Бэкенд | Нет (фронтенд-прототип) |

## Быстрый старт

```bash
# Вариант 1: Просто откройте файл в браузере
open grant.html

# Вариант 2: Локальный HTTP-сервер
npx serve .
# или
python3 -m http.server 8000
```

Перейдите на `http://localhost:8000/grant.html`

Дополнительная страница справки для пользователей:
- `http://localhost:8000/faq.html`

## Актуальные бизнес-правила

- На 3-м неодобрении заявка переводится в `postponed`.
- Автоматического возврата из `postponed` нет.
- После истечения 3 месяцев разблокировка выполняется только вручную Фасилитатором.
- После разблокировки заявка переходит в `fac_revision` и идет на повторную подачу.
- Документный пакет заявки: версионируется только Word-файл бизнес-плана (V1, V2, ...).
- PDF бизнес-плана и фото-комплект являются фиксированными вложениями (базовая версия от Фасилитатора).
- Для первичной отправки требуется полный комплект: Word + PDF + ровно 4 фото.
- Если Комитет отклоняет заявку, она возвращается в `gmc_revision` (в ШИГ / КУГ) с обязательным комментарием причины.
- При возврате из Комитета счетчик `revisionCount` (лимит 3 попыток у Фасилитатора) не сбрасывается.
- Для заявки в статусе `approved` Фасилитатор загружает подписанный скан договора о гранте.
- Для `approved` доступна отдельная full-width демо-форма создания текста договора (аккордеон в истории/финальных документах).
- Формат номера договора в системе: `Ш-******-ДДММГГ`.
- Одобренные и отклоненные заявки можно открывать в режиме только просмотра из любой роли.
- В `approved` Фасилитатор может прикрепить подписанный Шартнома (PDF/JPG/JPEG/PNG), запись отражается в истории, а скачивание отдает исходный загруженный файл.

## Актуальные изменения UI (технически)

- В разделе `approved_registry` поиск по ID/ФИО показывает карточку конкретного заявителя.
- Поиск по номеру списка/протокола показывает карточку списка.
- При пустом поиске отображаются карточки списков одобрений.
- В карточках и таблицах заявок показывается явный индикатор `Current Word Version: Vn`.
- В модальных блоках и таблице Комитета доступны отдельные кнопки скачивания: Word, PDF, фото-комплект.
- В разделе истории для `approved` заявки доступен блок подписанного договора: загрузка (Фасилитатор), метаданные файла, запись в audit log и скачивание исходного файла.

## Стандарт терминов в документации

- Роль оценки: `ШИГ / КУГ` (единое написание через пробелы вокруг `/`).
- Статус блокировки: `postponed`.
- Статус после ручной разблокировки: `fac_revision`.
- Признак после истечения срока блокировки: `unlock-ready` (готова к разблокировке).

## Языки интерфейса

Интерфейс билингвальный: **таджикский** (основной) + **русский** (дублирующий).  
Все метки содержат оба языка: `<span class="ru">/ русский текст</span>`
