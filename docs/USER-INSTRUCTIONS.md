# Пользовательские инструкции / User Instructions

## Назначение

Документ описывает практические шаги работы по ролям: Фасилитатор, ШИГ / КУГ, ГТЛ / ГРП, Комитет.

## Общие правила

- Интерфейс двуязычный: таджикский текст крупнее, русский меньшим шрифтом.
- Все действия фиксируются в журнале заявки.
- После каждого действия карточки и таблицы обновляются автоматически.

## Фасилитатор

### 1) Создать новую заявку

1. Нажмите кнопку `Дархости нав / Новая заявка`.
2. Найдите бенефициара по ФИО, ИНН или телефону.
3. Выберите запись в выпадающем списке.
4. Нажмите `Эҷод кардан / Создать заявку`.
5. В карточке заполните поля вкладки Фасилитатора.

### 2) Сохранить черновик

1. Заполните обязательные поля.
2. Нажмите `Захира / Сохранить черновик`.
3. Если данные бенефициара неполные, заявка перейдет в статус неполных данных до исправления.

### 3) Отправить в ШИГ / КУГ

1. Убедитесь, что обязательные поля заполнены.
2. Приложите документный пакет:
3. Word файл бизнес-плана.
4. PDF файл бизнес-плана.
5. Ровно 4 фотографии.
6. Нажмите `Ба ШИГ фиристодан / Отправить в КУГ`.
7. Заявка переходит в статус рассмотрения ШИГ / КУГ.

Важно:
- Word версия увеличивается только при новых корректировках документа.
- PDF и фото считаются фиксированными вложениями.

### 5) Работа со статусом На доработке от ШИГ (`fac_revision`)

1. Откройте заявку в статусе `На доработке`.
2. Обратите внимание на подсказку в блоке документов: PDF и фото зафиксированы.
3. Загрузите только обновленную Word-версию бизнес-плана.
4. Отправьте заявку повторно в ШИГ / КУГ.

Важно:
- Поля `Сектор` и `Сумма` в этом режиме фиксированы и не редактируются.
- Без новой Word-версии повторная отправка недоступна.

### 4) Разблокировать заявку после лимита

1. Откройте раздел отложенных заявок (`postponed`).
2. Если срок блокировки истек, появится кнопка `Разблокировать`.
3. Нажмите `Разблокировать` в карточке/таблице или в ленте уведомлений (колокольчик).
4. После разблокировки заявка перейдет в `fac_revision` и будет доступна для доработки и повторной отправки.

Важно:
- На 3-м неодобрении заявка автоматически уходит в `postponed`.
- Автоматического возврата через 3 месяца нет.
- Разблокировка возможна только ручным действием Фасилитатора.

## ШИГ / КУГ

### 1) Провести оценку

1. Откройте заявку в статусе рассмотрения.
2. Заполните критерии оценки и комментарии.
3. Выберите решение:
4. `Одобрить` — заявка идет дальше по процессу.
5. `Вернуть на доработку` — заявка возвращается Фасилитатору.

### 2) Лимит доработок

- Счетчик доработок увеличивается при каждом возврате на доработку.
- На третьем неодобрении заявка блокируется и переходит в `postponed`.

### 3) Возврат из ГТЛ / ГРП в ШИГ / КУГ

1. При статусе `gmc_revision` откройте заявку.
2. Загрузите новую Word-версию бизнес-плана.
3. Нажмите действие возврата в ГТЛ / ГРП.

Важно:
- Без новой Word-версии возврат в ГТЛ / ГРП не выполняется.

## ГТЛ / ГРП

1. Откройте заявку, направленную на проверку.
2. Выполните социально-экологическую оценку.
3. Заполните решение и комментарии.
4. Передайте результат в следующий этап процесса.

Дополнительно:
- В шапке заявки доступны отдельные кнопки скачивания: Word, PDF, фото.
- Word берется как текущая версия `Vn`, PDF/фото скачиваются как фиксированные вложения.

## Комитет

1. Откройте пакет/реестр заявок, готовых к рассмотрению.
2. Примите финальное решение по каждой заявке.
3. При необходимости выполните экспорт протокола.

Дополнительно:
- Для каждой заявки в таблице доступны три кнопки скачивания: Word, PDF, фото.

## Уведомления и метки

- Колокольчик в интерфейсе показывает заявки, готовые к ручной разблокировке.
- Для каждой записи доступны действия `Открыть`, `Разблокировать` (если доступно), `Обработано`.
- У заявок с ручной реактивацией отображается постоянная метка о возврате после периода блокировки.
- В карточках и таблицах отображается явный индикатор `Current Word Version: Vn`.

## Частые вопросы

### Почему не могу отправить заявку в ШИГ / КУГ?

Проверьте, заполнены ли обязательные поля и нет ли предупреждения о неполных данных бенефициара.

### Почему заявка все еще в postponed после истечения 3 месяцев?

Потому что автоматическая реактивация отключена. Нужна ручная разблокировка Фасилитатором.

### Как понять, что заявка в состоянии unlock-ready?

Появляется индикатор готовности и запись в ленте уведомлений Фасилитатора.
